Polecenie:

grep -RIn "/chamber" . | wc -l

składa się z 3 części.

1. grep

grep = szuka tekstu w plikach.

Przykład:

grep "hello" test.txt

znajdzie wszystkie linie zawierające:

hello
2. Opcje -RIn
grep -RIn "/chamber" .
-R

Recursive.

Czyli:

szukaj we wszystkich podfolderach

Bez tego grep przeszuka tylko bieżący katalog.

-I

Ignore binary files.

Pomija np:

.png
.exe
.db

żeby grep nie próbował czytać binarek.

-n

Numer linii.

Pokazuje:

plik:linia:tekst

np:

./templates/nav.html:12:<a href="/chamber">
3. "/chamber"

To tekst którego szukasz.

grep znajdzie np:

href="/chamber"

albo:

fetch("/chamber")
4. .

Kropka oznacza:

szukaj od aktualnego katalogu

Czyli:

grep -RIn "/chamber" .

= przeszukaj cały projekt.

5. |

Pipe.

Przekazuje wynik jednego programu do drugiego.

Czyli:

A | B

oznacza:

weź wynik A i podaj go do B
6. wc -l

wc = word count.

-l = count lines.

Czyli:

wc -l

liczy liczbę linii.

Finalnie
grep -RIn "/chamber" . | wc -l

oznacza:

znajdź wszystkie wystąpienia "/chamber"
we wszystkich plikach projektu
i policz ile ich jest
Bez wc -l

Dostajesz szczegóły:

grep -RIn "/chamber" .

wynik:

./templates/nav.html:12:<a href="/chamber">
./static/js/app.js:55:fetch("/chamber")
Z wc -l

Dostajesz tylko liczbę:

7

czyli:

"/chamber" występuje 7 razy

grep -RIn "fetch(" static/js

-----------------------------------------------
To uporządkuje routing i unikniesz konfliktów.

1. Znajdź wszystkie stare odwołania

W Git Bash:

grep -RIn "chamber_testing" .

Szukaj takich miejsc:

href="/chamber_testing"
window.location = "/chamber_testing"
render_template("chamber_testing.html")
2. Co zostawić

W research_routes.py zostaw tylko:

@research_bp.route("/chamber")
def chamber_testing():
    return render_template("chamber_testing.html")
3. Co usunąć

Usuń z main_routes.py:

@main_bp.route("/chamber_testing")
def chamber_testing_alias():
    return render_template("chamber_testing.html")
4. Zamień stare linki
Jeśli znajdziesz:
href="/chamber_testing"

zamień na:

href="/chamber"
Jeśli znajdziesz:
window.location = "/chamber_testing"

zamień na:

window.location = "/chamber"
Jeśli znajdziesz:
redirect("/chamber_testing")

zamień na:

redirect("/chamber")
5. Sprawdź czy zostały stare ścieżki

Po poprawkach:

grep -RIn "chamber_testing" .

Powinno zostać tylko:

render_template("chamber_testing.html")

czyli nazwa pliku HTML jest OK — nie zmieniaj jej.

6. Finalny routing
main_routes.py

 /contact
research_routes.py

Cały HBOT:

/api/sessions

/api/during_merge

CTRL + C
python app.py

http://127.0.0.1:5000/chamber
